# NoSQL-хранилище скриптов и схем TUSUR.EMC

Программная система для централизованного хранения и управления XML-описаниями электрических схем и скриптами системы TUSUR.EMC.

## Что умеет система

*   **Загружает и анализирует** XML-файлы формата `.tcd`
*   **Извлекает и хранит** данные о компонентах схем: резисторы, конденсаторы, источники напряжения, узлы и другие элементы
*   **Ищет элементы** по названию, типу или значению в текущей схеме или во всех сохранённых схемах
*   **Показывает статистику** — автоматически подсчитывает количество элементов каждого типа в схеме
*   **Сохраняет схемы и результаты поиска** в базе данных для последующего доступа
*   **Предоставляет удобный интерфейс** с вкладками для просмотра, поиска и управления схемами

## Из чего состоит система

Проект включает два основных модуля и базу данных:

1.  **`xml_processor.py`** — обрабатывает XML-файлы, извлекает данные и работает с базой данных MongoDB.
2.  **`main.py`** — содержит графический интерфейс на PyQt5 для взаимодействия с пользователем.
3.  **`main_gui.ui`** — файл дизайна интерфейса, созданный в Qt Designer.
4.  **MongoDB** — документо-ориентированная база данных, в которой хранятся все схемы и результаты.

### Требования
*   Установленный **Python 3.8+**
*   Установленная и запущенная **MongoDB** (локально на стандартном порту 27017)
*   Установленные зависимости из `requirements.txt`

## Как пользоваться

1.  **Загрузите схему**: Нажмите кнопку «Загрузить XML» и выберите файл `.tcd`.
2.  **Просмотрите элементы**: Во вкладке «Просмотр схемы» отобразятся все компоненты.
3.  **Сохраните в базу**: Нажмите «Сохранить в mongo», чтобы добавить схему в постоянное хранилище.
4.  **Выполните поиск**: Введите запрос в поле поиска, выберите режим («в текущей» или «во всех схемах») и нажмите «Найти».
5.  **Просмотрите все схемы**: Перейдите во вкладку «Все схемы», чтобы увидеть список всех сохранённых документов.

## Структура базы данных (MongoDB)

*   **База данных**: `talgat_db`
*   **Коллекция `schemas`**: Хранит загруженные XML-схемы с метаданными и списком элементов.
*   **Коллекция `search_results`**: Хранит сохранённые пользователем результаты поиска.

## Технологии

*   **Python** — основной язык разработки
*   **MongoDB** — документо-ориентированная база данных
*   **PyQt5** — фреймворк для графического интерфейса
*   **PyMongo** — драйвер для работы с MongoDB из Python
*   **xml.etree.ElementTree** — стандартная библиотека для парсинга XML
